Build a multi-agent system that combines CrewAI's unified memory with Mem0's persistent long-term storage. This integration enables agents to retain knowledge across sessions, crews, and projects while CrewAI handles within-session context.

## Overview

In this guide, we'll build a research crew that:
1. Uses CrewAI's built-in `Memory` for within-session context sharing between agents
2. Leverages Mem0 to persist important facts across sessions and crews
3. Retrieves user-specific context from Mem0 to personalize agent behavior
4. Stores task outputs in Mem0 for long-term retention and retrieval

## Setup and Configuration

Install the necessary libraries:

```bash
pip install crewai crewai-tools mem0ai
```

Import required modules and configure the clients:

<Note>Remember to get your API key from the <a href="https://app.mem0.ai?utm_source=oss&utm_medium=integration-crewai" rel="nofollow">Mem0 Platform</a> and [OpenAI](https://platform.openai.com).</Note>

```python
import os
from mem0 import MemoryClient
from crewai import Agent, Task, Crew, Process

# Configuration
os.environ["MEM0_API_KEY"] = "your-mem0-api-key"
os.environ["OPENAI_API_KEY"] = "your-openai-api-key"

# Initialize Mem0 client for persistent cross-session memory
mem0 = MemoryClient()
```

## Store User Context in Mem0

Before running a crew, store relevant user information in Mem0 so agents can access it during task execution:

```python
def store_user_context(user_id: str, messages: list):
    """Store conversation history and preferences in Mem0 for cross-session retrieval."""
    mem0.add(messages, user_id=user_id)

# Store past interactions and preferences
messages = [
    {"role": "user", "content": "I'm a machine learning engineer working on NLP projects."},
    {"role": "assistant", "content": "Great! I'll keep that in mind for research recommendations."},
    {"role": "user", "content": "I prefer PyTorch over TensorFlow and use Hugging Face transformers daily."},
    {"role": "user", "content": "My current focus is on retrieval-augmented generation and embedding models."},

store_user_context("user_42", messages)
```

## Create Agents with Mem0-Enhanced Context

Retrieve relevant memories from Mem0 and inject them into agent backstories so they operate with full user context:

```python
def build_enriched_backstory(base_backstory: str, user_id: str) -> str:
    """Enrich an agent's backstory with relevant user context from Mem0."""
    memories = mem0.search(base_backstory, filters={"user_id": user_id})

    context = "\n".join(f"- {m['memory']}" for m in memories["results"])

    if context:
        return f"{base_backstory}\n\nKnown user context:\n{context}"
    return base_backstory


# Create a research agent with personalized context
researcher = Agent(
    role="AI Research Analyst",
    goal="Find and summarize the latest research papers relevant to the user's interests",
    backstory=build_enriched_backstory(
        "You are an expert AI research analyst who tracks the latest developments in ML and NLP.",
        "user_42",
    ),
    allow_delegation=False,
)

# Create a writer agent
writer = Agent(
    role="Technical Writer",
    goal="Transform research findings into clear, actionable summaries",
    backstory="You are a skilled technical writer specializing in AI and machine learning content.",
    allow_delegation=False,
)
```

## Define Tasks and Crew

Set up tasks and a crew using CrewAI's built-in `Memory` for within-session agent coordination:

```python
# Define tasks
research_task = Task(
    description="""Search for the latest papers and advancements in RAG and embedding models.
    Focus on practical techniques the user can apply in their PyTorch-based workflow.""",
    expected_output="A structured summary of 5 recent papers with key findings and practical takeaways.",
    agent=researcher,
)

summary_task = Task(
    description="""Take the research summary and write a concise brief.
    Highlight actionable insights for an ML engineer using PyTorch and Hugging Face.""",
    expected_output="A polished research brief with actionable insights.",
    agent=writer,
)

# Create crew with CrewAI's built-in memory for within-session context
crew = Crew(
    agents=[researcher, writer],
    tasks=[research_task, summary_task],
    process=Process.sequential,
    memory=True,  # CrewAI's built-in memory handles within-session context
)
```

## Run the Crew and Persist Results

Execute the crew and save important outputs to Mem0 for future sessions:

```python
def run_and_remember(user_id: str):
    """Run the crew and persist key findings to Mem0."""
    # Execute the crew
    result = crew.kickoff()

    # Store the final output in Mem0 for cross-session retention
    mem0.add(
        [{"role": "assistant", "content": str(result)}],
        user_id=user_id,
        metadata={"source": "crewai"},
    )

    print("Research Brief:")
    print(result)
    return result


if __name__ == "__main__":
    run_and_remember("user_42")
```

## Multi-Session Pattern

The real power of combining CrewAI with Mem0 is maintaining context across multiple crew runs. Here's a pattern for a production system:

```python
def run_crew_with_memory(user_id: str, task_description: str):
    """Run a crew that leverages both CrewAI and Mem0 memory."""
    # 1. Retrieve user context from Mem0
    memories = mem0.search(task_description, filters={"user_id": user_id})
    user_context = "\n".join(f"- {m['memory']}" for m in memories["results"])

    # 2. Create agents with enriched context
    agent = Agent(
        role="Personal Assistant",
        goal="Complete tasks with full awareness of the user's history and preferences",
        backstory=f"You are a helpful assistant.\nUser context:\n{user_context}",
    )

    # 3. Create task and crew with CrewAI's built-in memory
    task = Task(
        description=task_description,
        expected_output="A comprehensive response tailored to the user's needs.",
        agent=agent,
    )

    crew = Crew(
        agents=[agent],
        tasks=[task],
        process=Process.sequential,
        memory=True,
    )

    # 4. Execute
    result = crew.kickoff()

    # 5. Persist to Mem0 for next session
    mem0.add(
        [
            {"role": "user", "content": task_description},
            {"role": "assistant", "content": str(result)},
        ],
        user_id=user_id,
    )

    return result
```

## Key Features

1. **Cross-Session Persistence**: Mem0 stores important facts that survive across crew runs, sessions, and restarts.
2. **Personalized Agents**: Inject user-specific context from Mem0 into agent backstories for tailored behavior.
3. **Dual Memory Architecture**: CrewAI's built-in `Memory` handles within-session agent coordination while Mem0 provides long-term persistent storage.
4. **Scalable Retrieval**: Use Mem0's semantic search to pull relevant context regardless of when it was stored.

## Benefits

1. **Contextual Continuity**: Users can run multiple crews over days or weeks, and each crew picks up where the last one left off via Mem0.
2. **Personalization at Scale**: Store per-user preferences and history in Mem0, then inject them into any crew or agent on demand.
3. **No Lost Knowledge**: Important task outputs and decisions are persisted in Mem0, preventing information loss between sessions.
4. **Clean Separation**: CrewAI manages agent-to-agent context within a run; Mem0 manages user-level context across runs.

## Conclusion

By combining CrewAI's unified memory with Mem0's persistent storage, you get the best of both worlds: intelligent within-session agent coordination and durable cross-session knowledge retention. This pattern works especially well for personalized assistants, research agents, and any system where context should accumulate over time.
